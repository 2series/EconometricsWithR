2.5 Exercises

1. Correlation and (Non)linearity I

Consider the following estimated simple linear regression model

\[\widehat{medv_i} = 34.554 - 0.95\times lstat_i.\]

with medv (median house value) and lstat (percent of households with low socioeconomic status) being variables from the already known Boston dataset.

The lm()mobject of the above model is available as mod in your working environment. The package MASS has been loaded.

Instructions:

  • Compute the correlation between medv and lstat.

  • Plot medv against lstat and add the regression line of the estimated model. What do you notice?

library(MASS) mod <- lm(medv ~ lstat, data = Boston) # Compute the correlation between medv and lstat # Plot medv against lstat and add the regression line # Compute the correlation between medv and lstat cor(Boston$medv, Boston$lstat) # Plot medv against lstat and add the regression line plot(medv ~ lstat, data = Boston) abline(mod, col = "red") ex() %>% check_function("cor") %>% check_result() %>% check_equal() test_or(ex() %>% check_function("plot") %>% { check_arg(., "formula") %>% check_equal() check_arg(., "data") %>% check_equal() }, ex() %>% override_solution("plot(Boston$lstat, Boston$medv)") %>% check_function("plot") %>% { check_arg(., "x") %>% check_equal() check_arg(., "y") %>% check_equal() }, ex() %>% override_solution("plot(Boston$medv ~ Boston$lstat)") %>% check_function("plot") %>% check_arg("formula") %>% check_equal() ) ex() %>% check_function("abline") #%>% check_arg("reg") %>% check_equal() success_msg("Correct! Although the correlation coefficient suggests that medv and lstat are negatively linearly dependent, the relationship between them is clearly nonlinear. Hence a simple linear regression specification is inappropriate here.")

Hints:

  • You can use cor() to compute the correlation between variables.

  • As usual you can use plot() and abline() to visualize regression results.

2. Correlation and (Non)linearity II

In the previous exercise we saw that the correlation coefficient is not able to properly capture the functional form of a regression model in that it suggested by construction a linear form, whereas the true functional form seems to be nonlinear (e.g. exponential).

As an alternative consider the following nonlinear model specification

\[medv_i = \beta_0 + \beta_1\times\log(lstat_i) + u_i.\]

The package MASS has been loaded.

Instructions:

  • Conduct the regression from above and assign it to log_mod.

  • Visualize your results by drawing a scatterplot and adding the corresponding regression line. In comparison to the previous exercise what do you notice now?

library(MASS) # Conduct the regression and assign it to mod_log # Draw a scatterplot and add the corresponding regression line # Conduct the regression and assign it to mod_log mod_log <- lm(medv ~ log(lstat), data = Boston) # Draw a scatterplot and add the corresponding regression line plot(medv ~ log(lstat), data = Boston) abline(mod_log, col = "red") ex() %>% check_object("mod_log") %>% check_equal() test_or(ex() %>% check_function("plot") %>% { check_arg(., "formula") %>% check_equal() check_arg(., "data") %>% check_equal() }, ex() %>% override_solution("plot(log(Boston$lstat), Boston$medv)") %>% check_function("plot") %>% { check_arg(., "x") %>% check_equal() check_arg(., "y") %>% check_equal() }, ex() %>% override_solution("plot(Boston$medv ~ log(Boston$lstat))") %>% check_function("plot") %>% check_arg("formula") %>% check_equal() ) ex() %>% check_function("abline") #%>% check_arg("reg") %>% check_equal() success_msg("Correct! Although the relationship is not perfectly linear, using a log transformation considerably improves our model fit and seems to be a reasonable model specification.")

  • Use lm() to conduct a regression.

  • As usual you can use plot() and abline() to visualize regression results.

3. Optimal Polynomial Order - Sequential Testing

From the previous exercise recall the following model

\[medv_i = \beta_0 + \beta_1\times\log(lstat_i) + u_i.\]

We already saw that this model specification seems to be a reasonable choice. However we cannot be entirely sure, if a higher polynomial order may be more suited to describe the data. Therefore please do the following:

The packages AER and MASS have been loaded.

Instructions:

  • Determine the optimal order of the polylog model by the sequential testing approach. Assume a maximum polynomial order of \(r=4\) and a significance level of \(\alpha=0.05\). You are more or less free to solve this exercise as long as you use a for() loop. We recommend the following approach:
    1. Estimate a model, say mod, which starts with the highest polynomial order.
    2. Save the p-value (with robust standard errors) of the relevant parameter and compare it with the significance level \(\alpha\).
    3. If you cannot reject the null, repeat steps 1 and 2 for the next lowest polynomial order, otherwise stop the loop and print out the polynomial order.
  • Extract the \(R^2\) of the selected model and assign it to the variable R2.
library(AER) library(MASS) # Find the optimal polynomial order of the polylog model. # Extract the R^2 from the selected model and assign it to R2. # Find the optimal polynomial order of the polylog model. for(i in 4:1){ mod <- lm(medv ~ poly(log(lstat), i, raw = T), data = Boston) pval <- coeftest(mod, vcov = vcovHC)[(i+1), 4] if(pval < 0.05){ print(i) break } } # Extract the R^2 from the selected model and assign it to R2. R2 <- summary(mod)$r.squared ex() %>% check_for() ex() %>% check_object("R2") %>% check_equal() success_msg("Correct! Using the sequential testing approach we get an optimal order of r=2.")

Hints:

  • The index for the for() loop should start at 4 and end at 1.

  • You can use poly() inside of lm() as a generic way of incorporating higher orders of a certain variable in the model. Besides the variable, you have to specify the degree of the polynomial via the argument degree and set raw = TRUE.

  • You can use coeftest() together with the argument vcov to obtain p-values (with robust standard errors). Use the structure of the resulting object to extract the relevant p-value.

  • Use an if() statement to check whether the condition for no rejection is fulfilled and stop the for() loop with the help of break otherwise.

  • To obtain the \(R^2\), you can use summary() and extract it via $r.squared.

4. Estimated Effect of a Unit Change

Reconsider the polylog model from the previous exercise that was selected by the sequential testing approach. As this model is logarithmic and of quadratic form, we cannot simply read off the estimated effect of a unit change in lstat from the coefficient summary. Instead we have to compute it manually. To do so assume in the following that we are interested in the effect of an increase in lstat from 10% to 11%.

The selected polylog model is available as mod_pl in your working environment. The package MASS has been loaded.

Instructions:

  • Arrange the relevant data points in a data frame.

  • Use the new values to predict the corresponding values of medv.

  • Compute the expected effect with the help of the function diff().

library(MASS) mod_pl <- lm(medv ~ poly(log(lstat), 2, raw = T), data = Boston) # Set up the relevant data points. # Predict the corresponding values of medv. # Compute the expected effect of the change. # Set up the relevant data points. new_data <- data.frame(lstat = c(10, 11)) # Predict the corresponding values of medv. Y_hat <- predict(mod_pl, newdata = new_data) # Compute the expected effect of the change. diff(Y_hat) ex() %>% check_object("new_data") %>% check_equal() ex() %>% check_object("Y_hat") %>% check_equal() ex() %>% check_function("diff") %>% check_result() %>% check_equal() success_msg("Correct! The expected change in medv for an increase in lstat from 10% to 11% is about -1.16 (that is -1160$).")

Hints:

  • Note that the column name of the data points in the data frame has to coincide with the name of the regressor.

  • You can use predict() together with the new data points to obtain the relevant values of medv.

  • diff() expects a vector and computes the differences between all entries of this vector.

5. Interactions between Independent Variables I

Consider the following regression model

\[medv_i=\beta_0+\beta_1\times chas_i+\beta_2\times old_i+\beta_3(chas_i\times old_i)+u_i\]

where \(chas_i\) and \(old_i\) are dummy variables. The first one takes the value \(1\), if the Charles River (a short river in the proximity of Boston) passes through suburb \(i\) and \(0\) otherwise. The latter accounts for a high proportion of old buildings and is constructed as follows

\[\begin{align} old_i = & \, \begin{cases} 1 & \text{if $age_i\geq 95$},\ 0 & \text{else}, \end{cases} \end{align}\]

with \(age_i\) being the proportion of owner-occupied units built prior to 1940 in suburb \(i\).

The package MASS has been loaded.

Instructions:

  • Append the binary variable old to the dataset Boston.

  • Conduct the regression from above and assign it to mod_bb.

  • Obtain a coefficient summary of the model. How can you interpret the coefficient \(\hat{\beta}_3\)?
